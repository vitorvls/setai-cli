# Generate Documentation

## Objetivo
Gerar documentação técnica viva. Documentar módulos, componentes e decisões.

## Input
- Módulo/componente a documentar
- Código relacionado
- Decisões tomadas
- Contexto do projeto

## Instructions
Gere documentação técnica para este módulo abordando:

1. **Responsabilidade**
   - O que este módulo faz
   - Qual seu propósito
   - O que não faz

2. **Fluxo de dados**
   - Como dados entram
   - Como são processados
   - Como saem

3. **Dependências**
   - O que este módulo depende
   - O que depende deste módulo
   - Integrações externas

4. **Decisões importantes**
   - Por que foi implementado assim
   - Trade-offs considerados
   - Alternativas descartadas

## Constraints
- Escreva como documentação interna para o time
- Evite linguagem genérica
- Seja específico e técnico
- Inclua exemplos quando útil

## Output
Documentação técnica completa com:
- Responsabilidade clara
- Fluxo de dados documentado
- Dependências mapeadas
- Decisões explicadas
