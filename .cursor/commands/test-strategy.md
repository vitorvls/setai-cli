# Test Strategy

## Objetivo
Estruturar testes antes ou junto do código. Definir estratégia de testes para um módulo ou feature.

## Input
- Módulo/feature a ser testado
- Código existente (se aplicável)
- Requisitos funcionais
- Regras de negócio

## Instructions
Você é responsável pela estratégia de testes do projeto.

Para este módulo:

1. **Liste cenários críticos**
   - Fluxos principais
   - Casos de uso essenciais
   - Integrações críticas

2. **Defina testes unitários necessários**
   - Funções a testar
   - Casos de teste por função
   - Cobertura esperada

3. **Defina testes de integração**
   - Integrações a testar
   - Fluxos end-to-end
   - Mocks necessários

4. **Aponte casos de borda e falhas esperadas**
   - Edge cases
   - Tratamento de erros
   - Validações de input

## Constraints
- Não escreva código ainda
- Foque em estratégia e planejamento
- Considere cobertura mínima (ver `.cursor/rules/testing-rules.md`)
- Seja específico sobre o que testar

## Output
Estratégia estruturada com:
- Cenários críticos
- Testes unitários planejados
- Testes de integração planejados
- Casos de borda identificados
